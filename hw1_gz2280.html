Problem 1b.¶

First allow me use my own words to define: i.i.d., Markov model and hidden Markov model:

1) i.i.d.

Independent outcomes which share a same distribution.

2) Markov model

Every state is only related to its previous one state, conditionally independent of other previous states.

3) Hidden Markov model

We observe the outcome of a system which is built on Markov model, but we aren't so sure about what's going on in some middle points.

Then check the gambler's fallacy based on a set of outcomes: $(X_1,X_2,X_3...X_n)$：

1) i.i.d.

False.

From the definition of gambler's fallacy we know that if "lucky streak" appears more often in the current, we expect "unlucky streak" appears more in the future. Thus $X_1,X_2,X_3...X_n$ would not be independent. Let's say,

$$ P(X_{100}="lucky"|X_{99}=X_{98}=...X_{1}="unlucky") > P(X_{100}="lucky") $$

Which against the rule of independency that they should be equal.

2) Markov model

False.

Gambler's fallacy could be a Markov model once we elimate "neutral" state. However, if there is a "neutral" state, it should be independent of "lucky" and "unlucky" and the Markov model is destroyed. Considering the following outcome sequence: (Denote "lucky" as $L$, "unlucky" as $U$, "neutral" as $N$, and subscript as the order of the experiment)

$$ L_1 \to L_2 \to U_3 \to L_4 \to N_5 \to L_6 \to U_7 $$

Clearly the beginning 4 experiments are Markov model, due to the fact that they are conditionally independent of previous states. However, from 5th experiment to 6th experiment, we could not find a relation of "netural" and "lucky". Gambler's fallacy never states the dependency of being "netural" and being "lucky", which allows us assume that they are independent. So from 5th to 6th, the Markov model breaks.

3) Hidden Markov model

False.

We could not find a trasition matrix from our situation. As we stated above, "neutral" is indepent of "lucky" or "unlucky".

import numpy as np
from scipy import linalg

def GSAMPLE(mu, Sigma, n):
  
  d = np.shape(mu)[0]  # dimension
  normal_mean, normal_sigma = np.zeros(d), np.diag(np.ones(d))  # parameters for generate g
  vec_g = np.random.multivariate_normal(normal_mean,normal_sigma,n)  # g is a n by d matrix: n samples, d variables
  mat_D, mat_V = linalg.schur(Sigma)  # Schur decomposition
  sqrt_D = linalg.sqrtm(mat_D)  # sqrt of matrix D
  
  G = np.dot(np.dot(mat_V,sqrt_D),np.transpose(vec_g))
  G = np.transpose(G)+mu
  
  return(G)

Observed rules: (For data sampling from standard normal dirstribution)

  • (plot1 to plot2) If we fix the number of bins of a histogram and increasing the volume of sample size, we'd get a graph which is closer to normal distribution, thus a better estimate.

  • (plot3, plot4 to plot5) If we fix the size of samples and increasing the bins, we would observe a smoother fit, but it would become rough again if the bin number keeps increasing. This process could be considered as the ratio $\frac{n}{N_{bins}}$ is decreasing. As the ratio decreases, some points in the middle give us the best fit.

  • (plot6) If our bin number is greater than the sample size, our graph may look weird since bars are discrete. This is true that we have to distribute limited samples to a broader range. This does not seem to be a good fit and normal distribution can be rarely observed.
